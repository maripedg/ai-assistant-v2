PCM_OP_SUBSCRIPTION_RERATE_REBILL

Component financial
Visibility public
Transaction required

Description

This opcode rerates events for a specified account. Events from a specified start time up to the current time are rerated. Rerating results in creation of adjustment events with storable class type /event/billing/adjustment/event. For usage events the adjustment represents the difference between the original rating and the new rating. For system events such as cycle events or purchase events the adjsutment event represents the negation of the original event. Rerating will generate new system events as the result of the new rating.

0 int    FLAGS In Out
Use optional

Flags value to indicate whether to rerate events based on the ordering of the END_T of the event or CREATED_T of the event. Default is END_T. Flags are defined PIN_BILL_RERATE_BY_END_T 0x01 PIN_BILL_RERATE_BY_CREATED_T 0x02 (Defined in pin_bill.h). In the case of multiple account rerating, only the PIN_FLD_FLAGS on the top level will be taken into account.

0 poid    SESSION_OBJ In Out
Use optional
Class /event/session

This field identifies the session object associated with this call to rerating. If this field is passed the adjustment events created by rerating will have their session object set to the input value. In case of rerating with price overrides an object of type /rerate_session/override_info will be created using the input value as the session object. The /rerate_session/override_info is used to identify cases where price overrides have been applied to an account.
