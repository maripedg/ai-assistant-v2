PCM_OP_SUBSCRIPTION_SET_BUNDLE

Component fm_subscription
Visibility public
Transaction required

Description

This opcode creates or modifies /purchased_bundle and its attributes. Upgrade(adding offerings to the existing /purchased_bundle) and downgrade(removing offerings from an existing /purchased_bundle) is also handled by this opcode.

Input details

0 poid    POID In Out
Use required
Class /account

Poid of the account owning the Bundle.

0 int    FLAGS In Out
Use optional

This flag is used at the time of bundle cancellation. If the value is set to PIN_SUBS_DELINK_BUNDLE_OFFERINGS(0x1), all offerings associated with the bundle to be cancelled will have their PIN_FLD_PLAN_OBJ set to NULL. If this flag is set, offerings associated with the bundle are not required in the opcode input.

0 string    PROGRAM_NAME In Out
Use required
Max length 60

Program name - identifies calling program.

0 array    BUNDLE_INFO In Out
Use optional
ElemID mode index
Iteration undefined
Min number of Elements 0

The promotion being purchased for the given account. List of purchased bundles that need to be created or modified. The element-id will be used to identify which offerings array to use from the input to associate this promotion bundle. In case of bundle creation: When validities are provided, PIN_FLD_VALID_FROM should be less than PIN_FLD_VALID_TO and PIN_FLD_VALID_FROM cannot be less than the owner account's minimum effective_t. If PIN_FLD_VALID_TO is a backdated date, PIN_FLD_VALID_FROM must be sent along with it and must be less than PIN_FLD_VALID_TO. If PIN_FLD_VALID_TO and PIN_FLD_VALID_FROM are send as NULL or nothing then default values set are VALID_FROM = immediately (current timestamp) and VALID_TO = never ends (0). If status field is not sent, it will be set to "ACTIVE". If PIN_FLD_ACCOUNT_OBJ is not passed, PIN_FLD_POID in the input will be set as the account obj. PIN_FLD_NAME is a mandatory field.
