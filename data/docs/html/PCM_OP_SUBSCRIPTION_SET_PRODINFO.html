PCM_OP_SUBSCRIPTION_SET_PRODINFO

Component fm_subscription
Visibility public
Transaction required

Description

PCM_OP_SUBSCRIPTION_SET_PRODINFO input and output flist specifications. This opcode is called to modify or set a product's purchase, cycle, or usage date information. If the PCM_OPFLG_CALC_ONLY flag is set, this opcode returns the entire event flist for the events created as a result of the modification. If not, it returns the event POIDs of all event objects created as a result of the modification. When automatic rerating is enabled, this opcode triggers automatic rerating of backdated product status changes when certain conditions are met.

2 int    FLAGS In Out
Use required
Parent in.PRODUCTS.OVERRIDE_PRODUCTS

This flags tells whether to create, read, update or delete the /offering_override_values. If the flag passed is 1(PIN_CREATE_DATE_RANGE), it will add new DATE_RANGE in the specified usage map and if that usage map doesn't exist, it will create new usage map. If no object is linked to purchased product, it will create a new object for overrides. If the flag passed is 2(PIN_UPDATE_TAGS), it will update or add the tag in the matching date ranges. If the flag passed is 3(PIN_DELETE_OFFER), it will delete the offering override object linked to purchased product. If the flag passed is 4(PIN_DELETE_DATE_RANGE), it will delete the matching date ranges. If the flag passed is 5(PIN_DELETE_TAGS), it will delete the listed tags in input for that date range.

Array of results information. Since only one event object is created by this call, only one element will be returned. The element-id is not significant. Fields of the event object. Additional event object fields are added to the return flist here. If the PCM_OPFLG_READ_RESULT flag is set for the operation, all the fields in the event object are returned on the output flist, not just the POID. If the PCM_OPFLG_CALC_ONLY flag is set, no fields in the database are changed and the event object is not actually created, but the fields that would have been used to create the event object are returned to the caller on the output flist.
